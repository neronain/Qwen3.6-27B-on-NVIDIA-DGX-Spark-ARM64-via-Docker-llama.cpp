เนื่องจากเครื่อง DGX Spark ของคุณมี VRAM เหลือเฟือ (128GB+) การปรับจูนเพื่อขยาย Context และดึงศักยภาพของ KV Cache ออกมาให้เต็มที่ ถือเป็นเรื่องที่ควรทำอย่างยิ่งสำหรับงานสาย Agent ที่ต้องอ่านโค้ดหรือเอกสารยาวๆ ครับ

ใน `llama.cpp` คุณสามารถเพิ่มพารามิเตอร์เหล่านี้เข้าไปในส่วน `command:` ของ `docker-compose.yml` ได้เลยครับ:

### 1. ขยายขนาด Context Window (`-c`)

ค่านี้คือความจำระยะสั้นของโมเดล (หน่วยเป็น Token) ตอนนี้คุณตั้งไว้ที่ `32768` (32K)

* **การปรับจูน:** โมเดลตระกูล Qwen และ LLM รุ่นใหม่ๆ มักจะรองรับ Context ได้สูงสุดถึง 64K หรือ 128K คุณสามารถปรับเพิ่มเป็น `-c 65536` (64K) หรือ `-c 131072` (128K) ได้เลย
* **ผลลัพธ์:** ทำให้ Agent สามารถอ่านไฟล์โปรเจกต์ขนาดใหญ่ โค้ดหลายพันบรรทัด หรือแชทคุยต่อเนื่องได้ยาวนานขึ้นโดยไม่ลืมบริบทเก่า (VRAM จะถูกจองเพิ่มตามขนาด Context ที่ตั้งไว้)

### 2. เพิ่มจำนวนการประมวลผลพร้อมกัน (Concurrent Slots / `-np`)

ค่า Default ของเซิร์ฟเวอร์มักจะตั้ง Slot ไว้ที่ 1 หรือ 4 หมายความว่าถ้ามี Agent ยิงคำถามเข้ามาพร้อมกันเยอะๆ มันจะต้องรอคิว

* **การปรับจูน:** เพิ่มพารามิเตอร์ `--parallel 8` (หรือ `-np 8`)
* **ผลลัพธ์:** ระบบจะแบ่ง KV Cache ออกเป็น 8 ช่องทาง ทำให้สามารถตอบคำถาม Agent 8 ตัวที่ยิงเข้ามาในวินาทีเดียวกันได้แบบขนาน (True Concurrency) ยิ่ง VRAM เยอะ ยิ่งเปิด Slot รับโหลดพร้อมกันได้มากครับ

### 3. เพิ่มความเร็วในการกลืน Context (Batch Size / `-b` และ `--ub`)

เมื่อคุณขยาย Context ใหญ่ขึ้น (เช่น โยนโค้ด 50,000 tokens เข้าไป) โมเดลจะต้องใช้เวลา "อ่าน" (Prompt Evaluation) ในช่วงแรก

* **การปรับจูน:** เพิ่มพารามิเตอร์ `-b 4096` และ `--ub 4096` (Physical & Logical Batch Size)
* **ผลลัพธ์:** เป็นการสั่งให้ GPU ค่อยๆ กลืน Token ทีละ 4,096 คำต่อรอบ (จากเดิมที่อาจจะแค่ 512) ด้วยพลังของการ์ดจอ Blackwell การเพิ่มค่านี้จะทำให้จังหวะที่โหลดโค้ดยาวๆ เร็วขึ้นแบบก้าวกระโดดครับ

---

### ตัวอย่างการอัปเดตไฟล์ `docker-compose.yml`

คุณสามารถแก้ส่วน `command:` ของโมเดลให้กลายเป็นแบบนี้ได้เลยครับ:

```yaml
    command: >
      -m /models/Qwen3.6-27B-UD-Q8_K_XL.gguf
      -ngl 999
      -fa on
      -c 65536                  # 1. ขยาย Context เป็น 64K
      --parallel 8              # 2. รองรับ Agent ทำงานพร้อมกัน 8 ตัวโดยไม่ต้องรอคิว
      -b 4096                   # 3. เร่งความเร็วการอ่าน Prompt ก้อนใหญ่
      --ub 4096                 # 3. ควบคู่กับ Logical Batch
      --reasoning-preserve
      --host 0.0.0.0
      --port 8080

```

> **💡 ข้อควรระวัง (Flash Attention):**
> สังเกตว่าเรายังคงเปิดพารามิเตอร์ `-fa on` (Flash Attention) ทิ้งไว้ **ห้ามลบเด็ดขาดนะครับ** เพราะอัลกอริทึมนี้คือหัวใจสำคัญที่ช่วยบีบอัดการใช้ VRAM ของ KV Cache เวลาที่เราขยาย Context ใหญ่ระดับ 64K+ หากไม่มี Flash Attention VRAM 128GB ก็อาจจะเต็มได้ครับ
